<h3 style="text-align:center;">ЭЦП на алгоритме RSA</h3>
<p>При описании данного алгоритма будем считать, что в его работе принимают участие банкир B и несколько вкладчиков W, W<sub>1</sub>, W<sub>2</sub>, ..., W<sub>n</sub>.</p>
<p>Подготовительный этап работы совпадает с аналогичным этапом для системы с открытым ключом. Именно, банкир B выбирает два простых числа p<sub>1</sub> и p<sub>2</sub>; находит их произведение R = p<sub>1</sub> ∙ p<sub>2</sub>; вычисляет φ(R) = (p<sub>1</sub> − 1) ∙ (p<sub>2</sub> − 1); выбирает свой открытый ключ S из условий S ∈ ℕ, (S, φ(R)) = 1, 0 < S < φ(R); находит свой секретный ключ T из условий T ∈ ℕ, S ∙ T ≡ 1 (mod φ(R)), 0 < T < φ(R).</p>
<p>Аналогичные действия выполняет каждый из вкладчиков W, W<sub>1</sub>, W<sub>2</sub>, ..., W<sub>n</sub>.</p>
<ol>
<li>
<p>Пусть m, 0 < m < r, - числовой эквивалент сообщения, составленного вкладчиком W для банкира B. (Убедимся, что, кроме того, выполняется и ограничение 0 < r < R.) Вкладчик W шифрует сообщение m секретным ключом t:</p>
<p>SW: m<sub>1</sub>≡m<sup>t</sup> (mod r), 0 < m<sub>1</sub> < r;</p>
<p>полученный результат m<sub>1</sub> он шифрует открытым ключом банкира S:</p>
<p>OB: m<sub>2</sub>≡m<sub>1</sub><sup>S</sup> (mod R), 0 < m<sub>2</sub> < R;</p>
<p>Полученное сообщение m<sub>2</sub> вкладчик W отправляет банкиру B.</p>
</li>
<li>
<p>Банкир B преобразует сообщение m<sub>2</sub> своим секретным ключом T:</p>
<p>SB: m<sub>3</sub>≡m<sub>2</sub><sup>T</sup> (mod R), 0 < m<sub>3</sub> < R;</p>
<p>а затем расшифровывает полученный результат m<sub>3</sub> открытым ключом вкладчика W:</p>
<p>OW: m<sub>4</sub>≡m<sub>3</sub><sup>s</sup> (mod r), 0 < m<sub>4</sub> < r.</p>
<p>В этом случае m<sub>4</sub>=m, и банкир B получает возможность прочитать исходное сообщение от вкладчика W, переводя число m в буквенную форму.</p>
</li>
</ol>
<ol>
<p>Замечание.
При доказательстве корректности работы алгоритма мы существенно использовали ограничение <i>r &lt; R</i>. При данном ограничении схема работы алгоритма выглядела так: SW, OB, SB, OW. Если же <i>R &lt; r</i>, то порядок действий будет иным: OB, SW, OW, SB. (Последовательность работы выбирается так, чтобы сравнения по большему модулю были внутри.)</li>
</p>
